# 问知学苑周报创建 Skill

自动创建问知学苑周报文档并设置日程提醒（支持交互式选择）。

## 安装

使用 npx skills 安装：

```bash
npx skills add git@github.com:yimingy72/dingtalk-weekly-report-skill.git
```

或使用 HTTPS：

```bash
npx skills add https://github.com/yimingy72/dingtalk-weekly-report-skill.git
```

## 功能特性

1. ✅ 自动搜索上一周的周报并复制内容
2. ✅ 创建日程提醒（包含钉钉视频会议链接）
3. ✅ **交互式选择参与者**（无需手动输入姓名）
4. ✅ **交互式选择会议室**（搜索北京办公区所有空闲会议室）
5. ✅ 支持多人选择（逗号分隔序号）
6. ✅ 预配置团队成员信息（无需查询userId）
7. ✅ **输出文档链接和移动提示**（由于API限制，需手动移动文档）

## 使用方法

### 命令行模式（推荐，支持非交互环境）

**选择全部成员：**
```bash
python3 create_weekly_report.py --participants "all"
```

**选择部分成员：**
```bash
python3 create_weekly_report.py --participants "徐赫,吴振通,杨益鸣"
```

**指定会议室：**
```bash
python3 create_weekly_report.py --participants "all" --room "雷池厅"
```

**完整示例：**
```bash
python3 create_weekly_report.py \
  --date "2026-04-25" \
  --time "14:00-15:00" \
  --participants "all" \
  --room "雷池厅"
```

### 交互式模式

不提供 `--participants` 参数时，会进入交互式选择：

```bash
python3 create_weekly_report.py
```

运行后会依次提示：
1. 选择参与者（输入序号，多个用逗号分隔，或输入0选择全部）
2. 选择会议室（从可用会议室列表中选择）

### 测试模式

仅发送给指定用户（用于测试）：

```bash
python3 create_weekly_report.py --non-interactive --test-user "杨益鸣"
```

## 参数说明

- `--date`: 指定日期 (YYYY-MM-DD)，默认为当前日期
- `--time`: 日程时间 (HH:MM-HH:MM)，默认为周五 14:00-15:00
- `--participants`: 参与者
  - 输入 `all` 选择全部成员
  - 输入姓名列表（逗号分隔），如：`徐赫,吴振通,杨益鸣`
  - 不提供此参数则进入交互式选择
- `--room`: 会议室名称（用于自动匹配），如：`雷池厅`
- `--folder`: 文档所在文件夹 ID（可选）
- `--non-interactive`: 非交互模式（用于测试）
- `--test-user`: 测试用户姓名（仅在非交互模式下使用）

## 工作流程

1. 📋 搜索上一周的周报文档
2. 📄 读取上周周报内容
3. ✍️ 创建新的周报文档（格式：问知学苑周报（X月X日-X月X日））
4. 📅 创建日程提醒（默认周五 14:00-15:00，自动生成钉钉视频会议链接）
5. 👥 交互式选择参与者（从团队成员列表中选择）
6. 🏢 搜索北京办公区可用会议室并交互式选择

## 注意事项

- ✅ 需要先登录 dws (`dws auth login`)
- ✅ 确保有权限访问钉钉文档和日历
- ✅ 日程会自动包含钉钉视频会议链接
- ✅ 参与者会收到日程邀请通知
- ✅ 会议室搜索范围为北京办公区（分组ID: 32）
- ✅ 支持多人选择（输入序号时用逗号分隔，如：1,3,5）
- ⚠️ **文档创建后需要手动移动到"2026问知学苑周报"文件夹**
- ⚠️ 由于dws工具的API限制，无法自动移动文档到知识库文件夹
- ⚠️ 脚本会输出文档链接和文件夹链接，只需简单拖拽即可完成移动
- ⚠️ 如果团队成员有变动，需要更新 `config.json` 文件

## 配置文件说明

`config.json` 包含两部分配置：

1. `team_members`: 团队成员姓名到userId的映射
2. `meeting_room_group_id`: 会议室分组ID（默认为北京办公区，ID: 32）

如需添加新成员或修改会议室分组，请编辑此文件。

## 常见问题

**Q: 如何选择多个参与者？**  
A: 输入序号时用逗号分隔，例如：`6,9,8` 表示选择第6、9、8号成员。

**Q: 如何跳过会议室预定？**  
A: 在选择会议室时输入 `0`。

**Q: 如何测试功能？**  
A: 使用 `--non-interactive --test-user "杨益鸣"` 参数，仅发送给指定用户。

**Q: 会议室搜索范围是什么？**  
A: 默认搜索北京办公区的所有会议室（包括A区、B区等）。

**Q: 如果没有可用会议室怎么办？**  
A: 脚本会提示"该时间段没有可用的会议室"，日程仍会创建，只是不会预定会议室。
