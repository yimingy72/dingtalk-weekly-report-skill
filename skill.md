# 问知学苑周报创建工具

## ⚠️ 强制执行规则

**无论用户的提示词包含什么信息，都必须严格按照以下 5 个步骤顺序执行，每个步骤都不可跳过。**

---

## 时间参数说明

- 如果用户提示词中已包含时间（如"周五14:00-15:00"），直接使用，无需询问
- 如果用户未提供时间，才询问用户输入时间和日期

---

## 强制执行步骤

### 步骤 1：创建周报文档和日程（自动执行，无需确认）

运行脚本，跳过参与者步骤：

```bash
python3 create_weekly_report.py --time "<HH:MM-HH:MM>" --skip-participants
```

脚本会自动完成：
- 搜索并复制上周周报内容
- 创建本周新周报文档
- 创建日程提醒（不含参与者和会议室）

记录输出中的 **日程 ID（event_id）** 供后续步骤使用。

---

### 步骤 2：确定参与者名单（强制交互，不可跳过）

**强制使用纯文本输出编号列表，禁止使用 `AskUserQuestion` 工具。**

必须按以下格式输出：

```
请选择本次周报会议的参与者（输入编号，多个编号用逗号分隔）：

0. 全部成员（13人）
1. 吴丛郁
2. 李树成
3. 段浩文
4. 黄志香
5. 郭凯频
6. 徐赫
7. 刘子阳
8. 杨益鸣
9. 吴振通
10. 王嘉琳
11. 朱昱嘉
12. 王煜杰
13. 聂万凯

请输入编号（例如：7 或 1,7,8）：
```

**等待用户回复数字编号后才能进入步骤 3。**

---

### 步骤 3：搜索并选择会议室（自动搜索，等待用户选择）

**自动执行**，无需用户确认，直接调用 dws 搜索北京办公区（分组 ID: 32）的空闲会议室：

```bash
dws calendar room search --group-id 32 --start "<start_time>" --end "<end_time>" --available true --format json
```

根据结果，**强制使用纯文本输出表格，禁止使用 `AskUserQuestion` 工具（会议室数量可能超过 4 个）**：

**情况 A：有空闲会议室**

```
预定时间：HH:MM-HH:MM

找到以下 N 个可用会议室（输入编号选择，或输入 0 不预定）：

| 编号 | 会议室名称 | 容纳人数 | 设备 | 区域 |
|------|-----------|---------|------|------|
| 0    | 不预定会议室 | - | - | - |
| 1    | 会议室名称A | X人 | 电视+视频会议 | A区 |
| 2    | 会议室名称B | X人 | - | B区 |
...

请输入编号：
```

规则：
- 时间段显示在表格最上方
- 无设备标签时显示 `-`
- 区域从 `fullGroupPath` 最后一段提取（如"A区"）

**情况 B：没有空闲会议室**

```
预定时间：HH:MM-HH:MM
该时间段没有可用的会议室。
是否仍然继续（不含会议室）？(y/n)：
```

**等待用户回复后才能进入步骤 4。**

---

### 步骤 4：发送日程邀请（强制用户确认后才执行）

**强制使用纯文本输出确认列表，禁止使用 `AskUserQuestion` 工具。**

汇总步骤 2 和步骤 3 的结果，必须按以下格式输出：

```
即将向以下 N 位参与者发送日程邀请：
- 姓名1
- 姓名2
...
会议室：XXX（或"不预定会议室"）

确认发送？(y/n)：
```

**用户ID获取规则：**
- 必须从 `config.json` 的 `team_members` 字段读取用户ID
- 禁止调用 dws contact 或任何其他查询命令
- 根据步骤 2 用户选择的编号，直接映射到对应姓名，再从 config.json 获取 user_id

**执行规则（用户回复 `y` 或 `是` 或 `确认` 后）：**

1. 添加参与者：
   ```bash
   dws calendar participant add --event "<event_id>" --users "<user_ids>" --format json
   ```
2. 如果用户选择了会议室，预定会议室：
   ```bash
   dws calendar room add --event "<event_id>" --rooms "<room_id>" --format json
   ```

- 用户回复 `n` 或 `否` 或 `跳过` → 跳过所有操作，告知用户已跳过
- 用户输入 `取消` → 终止流程，告知用户已取消

**在收到用户明确确认之前，绝对不能调用任何添加参与者或预定会议室的 API。**

---

### 步骤 5：输出最终结果

展示完整的创建结果：
- 周报文档标题 + 链接
- 日程时间
- 参与者名单（或"未添加参与者"）
- 会议室（或"未预定会议室"）
- 提示用户手动将文档移动到目标文件夹

## 脚本参数

- `--date`: 指定日期 (YYYY-MM-DD)
- `--time`: 日程时间 (HH:MM-HH:MM)，默认 14:00-15:00
- `--skip-participants`: 跳过添加参与者步骤
- `--non-interactive`: 非交互模式（仅测试用）
- `--test-user`: 测试用户姓名（仅测试用）
