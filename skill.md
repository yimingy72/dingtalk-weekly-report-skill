# 问知学苑周报创建工具

## ⚠️ 强制执行规则

**无论用户的提示词包含什么信息，都必须严格按照以下 5 个步骤顺序执行，每个步骤都不可跳过。**

---

## 强制执行步骤

### 步骤 1：创建周报文档和日程（自动执行，无需确认）

运行脚本，跳过参与者步骤：

```bash
python3 create_weekly_report.py --time "<HH:MM-HH:MM>" --skip-participants
```

脚本会自动完成：
- 搜索并复制上周周报内容
- 创建本周新周报文档
- 创建日程提醒（不含参与者）

记录输出中的 **日程 ID（event_id）** 供后续步骤使用。

---

### 步骤 2：确定参与者名单（强制交互，不可跳过）

**必须使用 `AskUserQuestion` 工具**，向用户展示预配置的 13 位团队成员，让用户选择参与者。

预配置成员：吴丛郁、李树成、段浩文、黄志香、郭凯频、徐赫、刘子阳、杨益鸣、吴振通、王嘉琳、朱昱嘉、王煜杰、聂万凯

提问示例：
> 请选择本次周报会议的参与者：
> - 全部成员（13人）
> - 自定义选择

**等待用户回复后才能进入步骤 3。**

---

### 步骤 3：发送日程邀请（强制用户确认后才执行）

**必须使用 `AskUserQuestion` 工具**，向用户展示完整参与者姓名列表，明确询问是否确认发送邀请。

提问示例：
> 即将向以下 N 位参与者发送日程邀请：
> [逐一列出姓名]
> 是否确认发送？

- 用户确认 → 调用 dws 添加参与者：
  ```bash
  dws calendar attendee add --event "<event_id>" --users "<user_ids>"
  ```
- 用户拒绝 → 跳过，不执行任何添加操作，告知用户已跳过

**在收到用户明确确认之前，绝对不能调用任何添加参与者的 API。**

---

### 步骤 4：搜索会议室（强制交互，不可跳过）

调用 dws 搜索北京办公区（分组 ID: 32）当前时间段的空闲会议室：

```bash
dws calendar room search --group 32 --start "<start_time>" --end "<end_time>"
```

根据结果，**必须使用 `AskUserQuestion` 工具**处理以下两种情况：

**情况 A：有空闲会议室**
> 找到以下 N 个可用会议室：
> [列出会议室名称]
> 请选择一个会议室，或选择"不预定会议室"。

**情况 B：没有空闲会议室**
> 该时间段没有可用的会议室。
> 是否仍然发送日程提醒（不含会议室）？

- 用户选择会议室 → 调用 dws 预定
- 用户选择不预定 / 仍然发送 → 继续，不预定会议室
- 用户选择取消 → 终止流程，告知用户已取消

**等待用户回复后才能进入步骤 5。**

---

### 步骤 5：输出最终结果

展示完整的创建结果：
- 周报文档标题 + 链接
- 日程时间
- 参与者名单（或"未添加参与者"）
- 会议室（或"未预定会议室"）
- 提示用户手动将文档移动到目标文件夹

## 脚本参数

- `--date`: 指定日期 (YYYY-MM-DD)
- `--time`: 日程时间 (HH:MM-HH:MM)，默认 14:00-15:00
- `--skip-participants`: 跳过添加参与者步骤
- `--non-interactive`: 非交互模式（仅测试用）
- `--test-user`: 测试用户姓名（仅测试用）
